# rel2sql TPC-H output

Generated by `scripts/tpch_emit_sql.sh` (all queries, **with** `sql::ast::Optimizer`) or `task tpch:emit-sql:mvp` / `scripts/tpch_smoke.sh` (MVP Q11, Q17, Q18, Q19, Q21, **full mode only** — includes `tpch_common_defs.rel`).

**Why solo fails with `l_revenue` / `dec` not defined:** `*_solo.rel` is the query file alone. Helpers like `l_revenue` and `dec` live in `benchmarks/TPCH/rel/tpch_common_defs.rel` and are only prepended in **full** mode. The `-e tpch_edb.edb` file lists base-table arities only; it does not define those Rel rules.

| Path | Contents |
|------|----------|
| `sql/q{N}.sql` | SQL from rel2sql (`-u`, with `tpch_common_defs.rel`) |
| `sql/q{N}_solo.sql` | Same query without common defs |
| `rel/q{N}.rel` | Rewritten Rel input fed to the CLI |
| `rel/q{N}_solo.rel` | Query-only rewritten Rel |
| `err/q{N}.err` | stderr when translation fails |

Reference TPC-H SQL (dbgen-style) lives in [`../sql/`](../sql/) (e.g. `../sql/q11.sql`).

## Compare

```sh
# After: task tpch:emit-sql   (or scripts/tpch_emit_sql.sh 11)
diff -u benchmarks/TPCH/sql/q11.sql benchmarks/TPCH/out/sql/q11.sql
# Side-by-side in Cursor: open both files
```

This directory is gitignored except this README.

`TIMEOUT_SEC=0` disables the per-query limit in `scripts/tpch_emit_sql.sh` (default 60s).

Skip queries (e.g. Q16 can hang with disjunction `MergeWith` safety): `SKIP_QUERIES=16 task tpch:emit-sql` or `task tpch:emit-sql:no-q16`.

Raw translator output (no optimizer): `UNOPTIMIZED=1 task tpch:emit-sql`.
